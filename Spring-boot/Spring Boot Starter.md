# 🚀 Spring Boot Starter: Hello World

This guide covers the basics of creating a simple "Hello World" endpoint in a Spring Boot application, including core annotations, HTTP methods, and understanding network concepts like IP addresses and ports.

---

## ❓ Why do we need this?

To build web applications, our code needs a way to receive requests from the internet and send back responses. Spring Boot uses annotations and standard HTTP methods to simplify this process, allowing us to quickly expose our Java methods as accessible web endpoints.

---

## 📌 Spring Annotations

In Spring Boot, annotations (starting with `@`) are used to provide metadata and instructions to the Spring framework. 

### Defining an API Controller
If you create a standard Java class, Spring won't automatically know it's meant to handle web requests. You must explicitly tell Spring by using the `@RestController` annotation.

```java
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloWorldController {
    // API logic goes here
}
```

### Exposing an Endpoint
To map a specific HTTP request to a function (so the function executes when that URL is visited), you use mapping annotations like `@GetMapping`.

```java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloWorldController {

    @GetMapping("/hello")
    public String sayHello() {
        return "Hello World!";
    }
}
```

---

## 🌐 REST & Controllers

- **Controller**: This acts as the manager for your API. It receives the incoming user requests, processes them (often by calling other services), and returns the correct response.
- **REST**: Stands for **Representational State Transfer**. It is a standardized architectural style that defines how systems communicate over the web efficiently using standard HTTP methods.

---

## 🖧 HTTP Request Types

When building RESTful APIs, you use different HTTP methods to perform various operations on data (often referred to as CRUD: Create, Read, Update, Delete).

| Method | Action | Description |
| :--- | :--- | :--- |
| **GET** | Read | Retrieves data from the server. |
| **POST** | Create | Sends new data to the server to be created. |
| **PUT** | Update | Updates data. Requires sending the *entire* modified object (changed and unchanged data). |
| **PATCH** | Update | Partially updates data. You only send the specific fields that need to change. |
| **DELETE** | Delete | Removes data from the server. |

## 🏠 IP Address vs. Port Number

When communicating over a network, you need both an IP Address and a Port Number (e.g., `192.168.1.1:8080`).

> [!NOTE]
> **The Analogy**: 
> Think of the **IP Address** as the address of a large apartment building. 
> Think of the **Port Number** as the specific apartment number (house) inside that building.

- **IP Address**: Uniquely identifies a single device (computer, server, smartphone) on a network or the internet. Your internet service provider (ISP) assigns this to you.
- **Port Number**: Identifies the specific service or application running on that device. 
  - *Example*: You use the same IP address to connect your phone to the internet, but different apps use different ports to communicate (e.g., Instagram might use one port standard, while Spotify uses another).

## 📂 Project Structure

A typical Spring Boot project generated via Maven follows a standard directory structure:

### 1. Source Folders (`src/`)
- **`main/java`**: Contains all your core application code, including controllers, services, models, and the main application executable file.
- **`test`**: strictly reserved for testing code (e.g., unit tests, integration tests). It does not get included in the final production build.

### 2. Build Outputs
- **`target/`**: This folder is generated by Maven during the build process. It contains the compiled bytecode (`.class` files) and the final packaged application (e.g., the `.jar` file) that gets executed.

### 3. Configuration
- **`pom.xml`**: The Project Object Model file. This is the heart of a Maven project. It contains the full details about your project, including all its dependencies, plugins, and build configurations.

## 🔄 Workflow Summary Recap
To create a functioning API endpoint:
1. Create a Java class and annotate it with `@RestController`.
2. Write a function that returns the desired data.
3. Annotate the function with an HTTP method (e.g., `@GetMapping("/path")`) to expose it to the web.
4. Run the application (which compiles into the `target` folder and starts on a specific port, often `8080`).
5. Access the endpoint via your browser or an API client using `http://localhost:8080/path`.
